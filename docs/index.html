Five analyzers.
Zero config.

codediag auto-detects your stack and runs every check that matters. NestJS-first, expanding to Next.js and Express.

API

API Health

Discovers endpoints from NestJS decorators. Checks auth guards, DTO validation, Swagger docs, and return types.

{}

Security

Scans for hardcoded secrets, missing .gitignore entries, helmet, CORS misconfig, rate limiting, and password hashing.

pkg

Dependencies

Runs npm audit, checks for lock files, deprecated packages, engine specs, and essential scripts.

tst

Testing

Checks test file existence, framework detection, test-to-source ratio, e2e directories, and coverage config.

src

Structure

Validates README, linter config, strict mode, module organization, and .env.example presence.

One unified score.

Every analyzer contributes to a weighted total. Security counts most, because shipping vulnerable code is the worst bug.

Works in CI too.

Add one line to your pipeline. codediag exits with code 1 if the score drops below your threshold.

# GitHub Actions
- name: Code Diagnostics
  run: npx codediag scan . --ci --threshold 80
